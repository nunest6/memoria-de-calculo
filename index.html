Changelog
v1.4.0 — 2025-04-05
FEAT LALUR / LACS — aba completa para Lucro Real
· Parte A: adições (provisões, multas, brindes, PLR, outras)
· Parte A: exclusões (dividendos, reversões, equiv. patrimonial)
· Compensação de prejuízo fiscal com limite automático de 30%
· Aviso visual quando compensação é limitada (saldo a transportar)
· IRPJ: 15% + adicional 10% sobre excedente de R$ 20.000/mês
· PAT com limite automático de 4% do IRPJ e aviso de limitação
· SUDENE/SUDAM: input de base incentivada + % de redução (75%/50%)
· CSLL: 9% com retenção na fonte
· Parte B: controle de saldo de prejuízo fiscal acumulado
· Resumo: carga efetiva IRPJ+CSLL sobre o Lucro Líquido
· Aba visível somente para clientes com regime = "real"
· Aba posicionada ao lado de Faturamento
FIX Persistência — campos calculados não eram filtrados antes do save
· Criado CAMPOS_INPUT com os 129 campos de input mapeados por sheet
· extrairInputs() garante que só inputs vão ao banco (não computed)
· Campos calculados (irpj_15, total_debitos, etc.) não mais salvos
FEAT Persistência — Simples Nacional integrado ao store e ao banco
· snRB12, snFolha12, snAnexo, snFolhaMeses eram variáveis globais
soltas e nunca eram persistidas
· Criado store[ano][mes].simples com todos os campos do SN
· syncSnParaStore(): copia globais → store antes do save
· syncStoreParaSn(): restaura globais ao carregar do banco
· snCommit() e snSetAnexo() agora acionam agendarAutoSave()
FIX Persistência — _saveInFlight silenciava saves concorrentes
· Antes: save chegando durante outro em andamento era descartado
· Agora: reagenda para 500ms após o save atual terminar
FIX Persistência — load não restaurava o Simples Nacional
· syncStoreParaSn() chamado após popular o store no carregamento
v1.3.0 — 2025-04-05
FEAT Persistência no Supabase — tabela memorias_calculo
· Upsert por (cliente_id, ano, mes) via Prefer: resolution=merge-duplicates
· Auto-save com debounce de 2s após cada edição de campo
· Botão "Salvar" manual na barra de ações
· Indicador de status: Aguardando / Salvando / Salvo / Erro
· Auto-load ao ativar cliente: carrega o ano completo do banco
· Auto-load ao trocar ano: recarrega o banco para o novo ano
· Cache _mesesCarregados evita re-fetch ao navegar entre meses
· Reset completo do store antes de popular com dados do banco
(evita misturar dados de clientes diferentes)
· Meses carregados exibidos no status (Jan, Fev, ...)
FIX Persistência — trocar mês mostrava tela vazia
· render() não era mais chamado antes do load async terminar
· Ao trocar mês: verifica cache antes de fazer nova requisição
v1.2.0 — 2025-04-05
FIX Supabase — seleção de cliente não funcionava
· Causa: onclick usava JSON.stringify duplo embutido no HTML
quebrando com nomes contendo aspas ou caracteres especiais
· Fix: substituído por Map em memória + data-idx no elemento
· Função selecionarCliente() global removida
FIX Supabase — pesquisa retornava registros incorretos ("reuniões")
· Causa: filtro JSONB aninhado via query string do PostgREST
não funciona para dados.cadastro.razao_social
· Fix: busca local com filtrarClientes() sobre cache de 500 rows
· carregarListaClientes() filtra rows sem razao_social preenchida
· filtrarClientes() normaliza acentos (COMERCIO = COMÉRCIO)
FIX Supabase — ativarCliente() ignorava dados.cadastro
· Causa: lia campos diretamente do row (c.nome, c.cnpj)
mas schema real tem tudo dentro de dados.cadastro.*
· Fix: rowParaCliente() implementado conforme schema documentado
· renderDropdown() também atualizado para usar rowParaCliente()
FIX Supabase — detectarCampos() e camposCliente removidos
· Heurística de detecção de campos era código morto e causava
race condition com initSeletorCliente()
